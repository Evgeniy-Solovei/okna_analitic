# План разработки

## Этап 1. Backend и данные

1. Поднять Django + PostgreSQL.
2. Заполнить `.env` доступами Bitrix24 и ID полей.
3. Синхронизировать пользователей, воронки, стадии.
4. Синхронизировать лиды.
5. Синхронизировать сделки.
6. Проверить доступность истории стадий.
7. Рассчитать и сохранить первую дату входа сделки в `ЗЗ`.
8. Пересчитать `manager_daily_metrics`.

## Этап 2. BI

1. Подключить Metabase к PostgreSQL.
2. Дать Metabase read-only пользователя.
3. Собрать вопросы на view `bi_manager_daily_metrics`.
4. Собрать дашборд с фильтрами:
   - период;
   - менеджер;
   - направление.
5. Проверить числовые подписи и читаемость графиков.

## Этап 3. Приемка

1. Пройти тестовые сценарии из ТЗ.
2. Сверить контрольные периоды с Bitrix24.
3. Зафиксировать расхождения.
4. Настроить роли пользователей.
5. Передать инструкцию по эксплуатации.
